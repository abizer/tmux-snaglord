# Fork notes

Private reminders for the abizer/tmux-snaglord fork.

## Fork-only additions

Everything above `upstream/main` on `main`:

- `flake.nix` / `flake.lock` — reproducible Nix build
- `.github/workflows/cache.yml` — push nix build to abizer.cachix.org
- `.github/workflows/upstream-check.yml` — daily issue when upstream has new commits
- `dist-workspace.toml` + `.github/workflows/release.yml` — cargo-dist release pipeline
- `meta/FORK-NOTES.md` (this file)

None of these land in the published crate (`meta/` is excluded in
Cargo.toml; everything else lives outside `src/`).

## Keep `main` in sync with upstream

```sh
fork-sync   # fetch upstream, rebase main, force-push
```

The `upstream-check` workflow runs daily and opens an `upstream-sync`-labelled
issue when upstream has new commits. It auto-closes when we catch up.

## Submit a PR upstream

Cherry-pick the mergeable commit(s) to a branch off `upstream/main`, open the
PR from that branch. Keep fork-only paths (`.github/`, `flake.*`,
`dist-workspace.toml`, `meta/`) off the PR branch.

## Cut a release

```sh
just release-patch   # bump Cargo.toml + tag + push
```

Pushing a `v*` tag triggers `.github/workflows/release.yml` (cargo-dist):

1. Builds all 4 targets natively (macOS arm/x86, Linux arm/x86)
2. Creates a GitHub Release with tarballs, sha256, source, shell installer
3. Commits the Homebrew formula to `abizer/homebrew-tap`

The workflow is generated by cargo-dist — **do not hand-edit it**. Change
`dist-workspace.toml` and run:

```sh
nix run nixpkgs#cargo-dist -- generate
```

Required repo secret: `HOMEBREW_TAP_TOKEN` (PAT with write to
`abizer/homebrew-tap`).
